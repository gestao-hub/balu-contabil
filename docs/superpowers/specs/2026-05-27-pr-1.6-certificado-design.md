# PR 1.6 — Configurações · Certificado A1 (design)

**Data:** 2026-05-27
**Status:** aprovado (brainstorming) — pronto para writing-plans
**Branch:** `feat/pr-1.6-certificado`
**Fontes:** PRD §8 (Configurações); infra existente `supabase-storage.ts`, `n8n.ts`, tabela `arquivos_auxiliares`.

## Contexto

Última aba da trilha de config fiscal. O usuário envia o **certificado digital A1** (`.pfx`/`.p12`) + senha; o sistema guarda o arquivo no Supabase Storage, registra em `arquivos_auxiliares` (caminho + senha) e **notifica o n8n** (que processa o certificado e o registra no provedor/Focus). Desbloqueia emissão de NFS-e para municípios que exigem certificado.

A infra já existe:
- `supabase-storage.uploadCertificado(file: ArrayBuffer|Buffer, fileName, companyId)` → `{ path }` (bucket `company-certificates`, usa **service_role**).
- `n8n.uploadCertificado({ unique_id_empresa, unique_id_bubble, file_base64, cert_password })` (HMAC via `N8N_WEBHOOK_SECRET`).
- `arquivos_auxiliares`: `id, unique_id_bubble, unique_id_empresa, supabase_file_path, storage_key, cert_password, created_at, updated_at, deleted_at`.
- `empresas_fiscais.certificado_jwt/_access_token/_token_expiration` — populados pelo **workflow n8n** (fora do escopo desta PR).

## Decisões aprovadas
1. **Salva primeiro, n8n depois.** Grava Storage + `arquivos_auxiliares`; depois chama o n8n em try/catch. Falha do n8n → `{ ok:true, warning }` (certificado salvo, processamento pendente). Não perde o upload se o n8n estiver fora (como o token Focus em teste).
2. **UI de upload** (status read-only + área de envio com botão "Enviar"/"Substituir"), não o toggle Editar/Salvar/Cancelar — é upload de arquivo, não edição de campos.
3. **`unique_id_bubble` = UUID gerado** no 1º upload (reaproveitado em re-uploads).
4. **Senha write-only** (coletada, nunca retorna ao client). Mesma ressalva de RLS das credenciais NFS-e (ver memória `balu-nfse-credenciais-plaintext`).

## Escopo
- `uploadCertificadoAction(formData)` + validador puro do arquivo.
- `CertificadoForm` (status + upload).
- Wire na aba `certificado` da page.

### Fora de escopo
- Parsing da resposta do n8n para `certificado_jwt/_access_token/_token_expiration` (o workflow n8n escreve isso no banco; YAGNI aqui).
- Credenciais Serpro (`/webhook/post-autenticacao`).
- Validade/expiração do certificado (schema não guarda validade de forma simples; exibe `certificado_token_expiration` só se presente).

## Arquitetura

### 1. `src/lib/fiscal/certificado.ts` (novo) — validador puro
```ts
export type CertValidacao = { ok: true } | { ok: false; error: string };
export function validateCertificadoUpload(input: { name: string; size: number; senha: string }): CertValidacao
```
Regras: nome termina em `.pfx` ou `.p12` (case-insensitive); `size > 0` e `size <= 1_000_000` (1MB — certificados A1 são pequenos; alinhado ao limite default de Server Action do Next); `senha` não-vazia. Mensagens claras em pt-BR. Testável (vitest).

### 2. `src/app/(auth)/configuracoes/actions.ts` — `uploadCertificadoAction` (novo)
Assinatura: `(formData: FormData) => Promise<{ ok: true; warning?: string } | { ok: false; error: string }>`.
1. `file = formData.get('file')` (instância de `File`), `senha = String(formData.get('senha') ?? '')`.
2. `validateCertificadoUpload({ name: file.name, size: file.size, senha })` — retorna erro se inválido.
3. Resolve `user` + `current_company` (companyId). Sem empresa → erro.
4. `const buf = await file.arrayBuffer();` → `storageUploadCertificado(buf, file.name, companyId)` → `path`. (em try/catch → erro)
5. Upsert `arquivos_auxiliares` por `unique_id_empresa = companyId` (`deleted_at is null`): se existe, `update { supabase_file_path: path, cert_password: senha, updated_at }`; senão `insert { unique_id_empresa: companyId, unique_id_bubble: <existente ou crypto.randomUUID()>, supabase_file_path: path, cert_password: senha }`.
6. `n8n.uploadCertificado({ unique_id_empresa: companyId, unique_id_bubble, file_base64: Buffer.from(buf).toString('base64'), cert_password: senha })` em try/catch → em erro, `warning = 'Certificado salvo, mas o processamento (n8n) falhou — será reprocessado.'`.
7. `revalidatePath('/configuracoes')`; retorna `{ ok: true, warning }`.

> **Risco de RLS (verificar):** o write em `arquivos_auxiliares` usa o client da sessão (como as outras actions de config). `arquivos_auxiliares` não tem `owner_user_id` e tem `SELECT(cert_password)` revogado (migration 0001). Se o insert/update bater em RLS, trocar o write por client **service_role** (padrão do `supabase-storage.ts`). Decidir na verificação.

### 3. `src/app/(auth)/configuracoes/CertificadoForm.tsx` (novo, client)
- Props: `enviadoEm: string | null` (created_at/updated_at do registro de certificado, ou null).
- **Status** read-only: se `enviadoEm` → "Certificado enviado em {data}"; senão "Nenhum certificado enviado".
- **Upload**: `<input type="file" accept=".pfx,.p12">` + `<input type="password">` (senha) + botão **"Enviar certificado"** (label "Substituir certificado" se `enviadoEm`).
- Submit (client handler): monta `FormData` (`file`, `senha`) → `uploadCertificadoAction(fd)`; `busy` no botão; toast: sucesso → "Certificado enviado."; sucesso com `warning` → toast `warning` com a mensagem; erro → toast erro. Limpa os campos no sucesso.
- Senha nunca exibida; o status não traz a senha.

### 4. `src/app/(auth)/configuracoes/page.tsx` — wire
- Quando `tab==='certificado'` e há `company`: consultar `arquivos_auxiliares?unique_id_empresa=eq.{companyId}&deleted_at=is.null&select=created_at,updated_at` (**sem** `cert_password`); derivar `enviadoEm = updated_at ?? created_at ?? null`.
- Renderizar `<CertificadoForm key={company.id} enviadoEm={...} />` no branch `certificado` (substitui o `TodoPanel`).

## Fluxo de dados
`page.tsx` (server) → status do certificado (read) → `<CertificadoForm>` → submit (FormData) → `uploadCertificadoAction` → valida → Storage + `arquivos_auxiliares` → n8n (best-effort) → `revalidatePath` → toast.

## Tratamento de erro
- Arquivo inválido/senha vazia → erro do validador (toast).
- Sem empresa → "Nenhuma empresa selecionada."
- Falha de Storage → erro (upload não persiste).
- Falha de `arquivos_auxiliares` → erro (repassa `error.message`).
- Falha de n8n → **não** falha o upload; `{ ok:true, warning }` (toast de aviso).

Confirmar no banco/Storage; confirmar RLS (item acima).

## Premissas documentadas
- Certificados A1 são pequenos (≤1MB) — limite alinhado ao default de Server Action do Next.
- `unique_id_empresa` = `companyId` (current_company); `unique_id_bubble` = UUID gerado.
- A senha (`cert_password`) é gravada sem criptografia/RLS hoje (carry-forward, ver [[balu-nfse-credenciais-plaintext]]).
- Os campos `certificado_*` em `empresas_fiscais` são preenchidos pelo workflow n8n, não por esta PR.
