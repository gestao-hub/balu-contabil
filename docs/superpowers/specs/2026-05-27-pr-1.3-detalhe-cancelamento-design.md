# PR 1.3 — Notas fiscais · Detalhe + cancelamento (design)

**Data:** 2026-05-27
**Status:** ✅ implementado e mergeado em `main` (`6eb49ce`, 2026-05-27). Migration `0004` aplicada no banco. Cancelamento/downloads exercitáveis quando o token Focus destravar (Blocked). Segue pendente o fix do webhook (`ref`→`referencia`) que popula os campos exibidos.
**Branch:** `feat/pr-1.3-detalhe-cancelamento` (mergeada e removida)
**Fontes:** PRD §10.1; kanban "PR 1.3" + "Passo 2"; infra existente `focus-nfe.ts`, `PopupConfirm`, listagem PR 1.2 (`notas_fiscais/page.tsx`, `actions.ts`).

## Contexto

`/notas_fiscais/[id]/page.tsx` hoje é um **stub** gerado por `gen_routes.py`. A listagem (PR 1.2) já existe e lê os dados do cliente de `payload_focusnfe.destinatario` (não há FK `cliente_id`). Quem popula `notas_fiscais` é o **webhook da Focus** (`src/app/api/webhooks/focus/route.ts`) — logo `payload_focusnfe` é a resposta/notificação da Focus. **Não há emissão escrevendo notas ainda** (isso é PR 2.1).

Tabela real `notas_fiscais` (db_atual.sql): `id, company_id, tipo_documento (NFe|NFCe|NFSe), referencia, data_emissao, status (pendente|ativa|cancelada), valor_total, payload_focusnfe (jsonb), created_at`. **Não tem** colunas de cancelamento nem de identificadores fiscais.

O cliente Focus já expõe o necessário: `focus.cancelarNfe/Nfce/Nfse(ref, justificativa, env)` (valida justificativa ≥15 e lança) e `focus.baixarDanfe/baixarDanfeNfce/baixarDanfeNfse` (PDF, ArrayBuffer) + `focus.baixarXmlNfe/Nfce/Nfse` (XML, string). **O token Focus está Blocked** ("não autentica") — então cancelamento e downloads só serão exercitáveis quando o token destravar (homologação).

## Decisões aprovadas
1. **Feature completa**: detalhe + downloads (XML/PDF) + cancelamento, todos via Focus. Métodos já existem no client; exercitáveis quando o token destravar.
2. **Migration "Passo 2" completa** (`0004`): adiciona as ~11 colunas de emissão+cancelamento de uma vez (prepara Day 2 também). PR 1.3 só **escreve** as de cancelamento; as de emissão ficam pra PR 2.1 popular.
3. **Download via route handler proxy** (não vaza o token Focus ao client).
4. **Detalhe lê com fallback pro `payload_focusnfe`** — `coluna ?? payload ?? '—'` — então funciona já com as linhas atuais (colunas novas = NULL).

## Escopo
- Migration `0004_notas_emissao_cancelamento.sql` + tipos em `database.ts`.
- `[id]/page.tsx` (server): detalhe.
- `[id]/CancelarButton.tsx` (client): confirmação + justificativa.
- `notas_fiscais/actions.ts`: `cancelarNotaAction(id, justificativa)`.
- `[id]/download/route.ts`: proxy de download XML/PDF.
- Helper puro `focusMethodForTipo` (dispatch por `tipo_documento`) — testável.

### Fora de escopo
- Popular as colunas de emissão (`chave_acesso`, `numero_nf`, `protocolo_autorizacao`, etc.) — é a PR 2.1 (emissão).
- FK `cliente_id` — coluna criada, mas o cliente segue vindo do `payload_focusnfe.destinatario`.
- Destravar o token Focus (item Blocked separado).
- Seleção de ambiente prod/homologação por empresa — usa `'hom'` por ora (default do client).

Atualizar o tipo `notas_fiscais` em `src/types/database.ts` com os 11 campos (`| null`). **A aplicação no banco é manual (humano).**

### 2. Helper puro `focusMethodForTipo` (em `notas_fiscais/actions.ts` ou módulo próprio)
Mapeia `tipo_documento` → as funções Focus corretas. Puro, sem `server-only`, testável:
```ts
export type TipoDoc = 'NFe' | 'NFCe' | 'NFSe';
export function assertTipoDoc(t: string): TipoDoc { /* lança se inválido */ }
// dispatch usado pela action e pelo route handler:
//   cancelar: NFe→focus.cancelarNfe, NFCe→focus.cancelarNfce, NFSe→focus.cancelarNfse
//   xml:      NFe→focus.baixarXmlNfe, NFCe→focus.baixarXmlNfce, NFSe→focus.baixarXmlNfse
//   pdf:      NFe→focus.baixarDanfe,  NFCe→focus.baixarDanfeNfce, NFSe→focus.baixarDanfeNfse
```
(A parte testável é `assertTipoDoc` + o mapeamento de nomes; as funções Focus em si são I/O.)

### 3. `[id]/page.tsx` (server component)
- Params `{ id }`. Resolve `user` + `current_company`. Busca:
  `notas_fiscais?id=eq.{id}&company_id=eq.{companyId}` (single). Ausente → `notFound()` (404).
- **Header:** `tipo_documento`, `referencia`, badge de `status`, `data_emissao` (pt-BR), `valor_total` (R$), cliente = `payload.destinatario.razao_social` + `cnpj/cpf`.
- **Identificadores:** `chave_acesso ?? payload.chave_nfe ?? '—'`, `protocolo_autorizacao ?? payload.protocolo ?? '—'`, `numero_nf`/`serie` (col ?? payload).
- **Se `status='cancelada'`:** bloco com `cancelled_at` + `cancellation_reason`.
- **Ações:**
  - "Baixar XML" / "Baixar PDF" → `<a href="/notas_fiscais/{id}/download?formato=xml|pdf">`.
  - `<CancelarButton id={id} disabled={status!=='ativa'} />` (client).
- Leitura defensiva: chaves exatas de `payload_focusnfe` confirmadas contra o webhook na implementação.

### 4. `[id]/CancelarButton.tsx` (client)
- Botão "Cancelar nota" (vermelho); desabilitado se `status!=='ativa'`.
- Abre `<PopupConfirm variant="destructive">` com `<textarea>` de justificativa (placeholder: "mín. 15 caracteres — regra SEFAZ").
- Valida ≥15 no client antes de habilitar o confirmar; `busy` durante a ação; `try/finally` reseta `busy` (lição do bug do seletor de empresa).
- Sucesso → toast + a revalidação atualiza a página; erro → toast com a mensagem da Focus.

### 5. `cancelarNotaAction(id, justificativa)` (server action)
1. auth + `companyId`. Sem empresa → erro.
2. Busca a nota (`id` + `company_id`). Ausente → erro. `status!=='ativa'` → erro ("Só notas ativas podem ser canceladas.").
3. `justificativa.trim().length >= 15` senão erro (mensagem amigável antes de bater na Focus, que também valida).
4. **Focus primeiro:** `focus.cancelar{...}(referencia, justificativa, 'hom')` por `tipo_documento`. Lança → retorna `{ ok:false, error }` e a nota **permanece ativa**.
5. **Só no sucesso:** PATCH `notas_fiscais` (`id`): `status='cancelada', cancelled_at=now, cancellation_reason=justificativa, updated_at=now`.
6. `revalidatePath('/notas_fiscais')` + `revalidatePath('/notas_fiscais/'+id)`. Retorna `{ ok:true }`.

### 6. `[id]/download/route.ts` (route handler)
- `GET ?formato=xml|pdf`. auth + busca nota (`id`+`company_id`) → 404 se ausente.
- Dispatch por `tipo_documento` + `formato`: `focus.baixarXml*` (string) ou `focus.baixarDanfe*` (ArrayBuffer), env `'hom'`.
- Responde com `Content-Type` (`application/xml` ou `application/pdf`) + `Content-Disposition: attachment; filename="{referencia}.{ext}"`.
- Erro da Focus → `Response` 502 com mensagem curta (não vaza token).

## Fluxo de dados
**Detalhe:** `page.tsx` (server) → SELECT nota (id+company_id) → render (colunas ?? payload).
**Cancelar:** `CancelarButton` → `cancelarNotaAction` → Focus DELETE → PATCH → revalidate.
**Download:** link → route handler → auth+SELECT → Focus baixar* → stream attachment.

## Tratamento de erro
- Nota inexistente / de outra empresa → `notFound()` (página) / 404 (route).
- `status!=='ativa'` no cancelar → erro claro; botão já vem desabilitado.
- Justificativa <15 → erro antes da Focus.
- Falha Focus (token Blocked, rede, SEFAZ) → erro exibido; **nota não é marcada cancelada**; downloads retornam 502.

## Verificação
- **vitest:** `focusMethodForTipo`/`assertTipoDoc` (mapeia NFe/NFCe/NFSe; lança em tipo inválido) + guarda da justificativa (≥15) como função pura testável.
- `tsc --noEmit` zero erros.
- **Manual (quando o token Focus destravar / homologação):** abrir uma nota → header/identificadores corretos; baixar XML/PDF; cancelar uma nota `ativa` com justificativa ≥15 → vira `cancelada` com `cancelled_at`/`cancellation_reason`; tentar <15 → erro; nota de outra empresa → 404.

## Premissas
- `env = 'hom'` (default do client; prod bloqueado). Quando produção entrar, derivar de `empresas_fiscais.producao_liberada`/`homologacao_liberada`.
